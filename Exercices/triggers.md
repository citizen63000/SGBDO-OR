# Sujet : Historisation des changements de salaire

Objectif : Créer un système d'audit automatique. Chaque fois qu'un employé reçoit une augmentation ou une baisse de salaire, Oracle doit automatiquement enregistrer l'ancienne valeur, la nouvelle valeur et la date de modification dans une table d'historique.

## Model de données
Le Modèle de Données

```SQL
CREATE TABLE employees (
    employee_id NUMBER PRIMARY KEY,
    name        VARCHAR2(50),
    salary      NUMBER(10,2)
);

-- Création de la table d'audit
CREATE TABLE salary_history (
    audit_id     NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    employee_id  NUMBER,
    old_salary   NUMBER(10,2),
    new_salary   NUMBER(10,2),
    changed_at   DATE DEFAULT SYSDATE
);

-- Insertion du jeu d'essai
INSERT INTO employees VALUES (1, 'Alice', 3000.00);
INSERT INTO employees VALUES (2, 'Bob', 2500.00);
COMMIT;
```

## Travail à faire
Créer un déclencheur (Trigger) nommé trg_audit_salary qui respecte les contraintes suivantes :

- Il doit se déclencher après (AFTER) chaque mise à jour (UPDATE) ciblant spécifiquement la colonne salary de la table employees.
- Il doit s'exécuter pour chaque ligne modifiée (FOR EACH ROW).
- Il doit insérer un enregistrement dans la table salary_history en exploitant les variables système :OLD.salary et :NEW.salary.

## Scénario de test
Pour valider le fonctionnement, on applique une augmentation de 500 $ à Alice, puis on observe si la table d'audit s'est remplie de manière invisible.
```SQL
-- 1. On applique la modification sur la table principale
UPDATE employees 
SET salary = 3500.00 
WHERE employee_id = 1;

COMMIT;

-- 2. On vérifie l'effet de bord automatique du trigger
SELECT * FROM salary_history;
```
